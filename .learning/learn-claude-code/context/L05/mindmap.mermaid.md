## 核心格言

> *"用到什么知识，临时加载什么知识"*
>
> Skill Loading 的本质：两层注入实现认知管理。
>
> 第一层是导航（可以放弃），第二层是本体（必须保留）。


## 情境测试要点

| 情境 | 核心风险 | 不变量提炼 |
|------|----------|------------|
| 资源约束反转 | 第一层被认为必需 | 第二层是本体，第一层是导航辅助 |
| 时序错乱 | 隐性行为误认为显性设计 | 确定性优于随机性 |
| 主体替换 | 容错性设计误认为偶然 | 宽容有边界 |
| 目标冲突 | 设计权衡误认为可逆转 | messages 隔离是物理约束 |
| 尺度折叠 | 两层注入价值误认为只省 token | 架构一致性 > 短期省钱 |
