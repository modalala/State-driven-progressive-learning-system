def persist_large_output(output):

def micro_compact(messages):
    tool_results = collect_tool_results(messages)
    for result in tool_results[:-3]:  # 只保留最近3个
        result["content"] = "[Earlier tool result omitted]"
    return messages

# Layer 3: 整体过长生成摘要
def compact_history(messages):
    summary = summarize(messages)  # 保住连续性5要素
    return [{"role": "user", "content": f"Compacted.\n{summary}"}]

## 关键概念速记

| 概念 | 一句话定义 | 记忆口诀 |
|------|-----------|----------|
| **三层压缩** | 大→磁盘、旧→占位、长→摘要 | "大存旧替长摘要" |
| **连续性5要素** | 目标/已完成/文件/决定/下一步 | "目完文决下" |
| **控制权归属** | 低成本Harness，高成本模型 | "低H高M" |
| **占位符** | 指针而非副本，指向不包含 | "指不包" |
| **推理链路优先** | 保住为什么，丢弃做了什么 | "保因弃果" |

## 核心原则（面试必答）

### 1. 压缩本质原则

**定义**：压缩是"可恢复损失 ↔ 可持续空间"的权衡，不是删除。

**关键点**：
- 所有压缩都是有损压缩，但损失可恢复
- 指针保留"可恢复性"，磁盘保留"内容"
- 不是"撞到上限才压缩"，而是三层主动管理

### 2. 连续性保护原则

**定义**：摘要必须保住连续性5要素，模型才能继续工作。

**5要素**：
1. 当前任务目标
2. 已完成的关键动作
3. 已修改/查看过的文件
4. 关键决定与约束（为什么选A不选B）
5. 下一步应该做什么

### 3. 控制权归属原则

**定义**：决策成本决定控制权归属。

| 决策成本 | 控制权 | 示例 |
|---------|--------|------|
| 低（固定规则） | Harness | Layer 1/2 触发阈值 |
| 高（时机判断） | 模型 | Layer 3 compact 工具 |

## 常见陷阱

| 陷阱 | 错误理解 | 正确理解 |
|------|----------|----------|
| 压缩=删除 | 把旧内容删掉腾空间 | 换表示方式保留关键信息 |
| 撞上限才压缩 | 只要不超过上限不需要 | 三层主动管理，预防式压缩 |
| 摘要=简洁 | 只给结论，省字数 | 保住推理链路，让模型能继续 |
| 颗粒度随意 | 压缩随便做，省空间最重要 | 推理链路优先 → 恢复成本 → 省空间 |
| read_file可压缩 | 和其他工具一样处理 | 环境漂移风险 + 特殊保护 |

## 自测问题

1. **环境漂移问题如何类比？什么是正确处理方式？**
   <details>
   <summary>点击查看答案</summary>
   
   **类比**：MySQL不可重复读——重新执行工具可能读到修改后的内容。
   
   **处理**：read_file特殊保护，不压缩，保留原结果。权衡：加锁（阻止修改）vs 备份（保留副本）。
   </details>

2. **摘要丢失"关键决策"会发生什么问题？举具体场景。**
   <details>
   <summary>点击查看答案</summary>
   
   **场景**：bug调试时，已确定是新语法特性而非编码问题，压缩后丢失决策。
   
   **后果**：模型偏向编码问题着手 → 重复排查错误方向 → 时间浪费。
   
   **本质**：决策记录保住"为什么"，动作记录只保"做了什么"。
   </details>

3. **占位符为什么是"指针"而非"副本"？**
   <details>
   <summary>点击查看答案</summary>
   
   **指针**：[Previous: used read_file] 只指向，不包含内容。
   
   **副本**：保留部分内容摘要。
   
   **为什么**：所有压缩都是有损压缩。指针保留"可恢复性"（全文在磁盘），副本丢失"可恢复性"（内容被截断）。
   </details>

4. **如何反驳"只要能干活，压缩随便做，省空间最重要"？**
   <details>
   <summary>点击查看答案</summary>
   
   **反驳逻辑**：压缩优先级 = 推理链路优先 → 恢复成本 → 省空间
   
   - 频繁压缩 → 恢复成本高（重新调用工具）
   - 决策丢失 → 方向偏离（重复错误）
   - 压缩本质是认知连续性保护，不是删历史
   </details>
