## 一、心智模型构建

### 1.1 核心概念网络

| 概念 | 定义 | 依赖关系 | 熟练度 |
|------|------|----------|--------|
| **权限管道** | tool_call → deny → mode → allow → ask | → Agent Loop | ✅ L2 |
| **管道顺序原则** | Deny优先→Mode→Allow→Ask兜底 | → 安全优先级 | ✅ L2 |
| **Bash特殊原则** | bash不是普通文本，需要独立前置检查 | → 危险命令过滤 | ✅ L2 |
| **三种模式** | default(问用户)、plan(只读)、auto(安全自动过) | → 会话风格控制 | ✅ L2 |
| **读写分类** | 判断工具/命令是读还是写操作 | → Mode层判断 | ⚠️ L1 |
| **存储过程边界** | 存储过程读写性质判断困难 | → 元数据标注 | ⚠️ L1 |
| **迁移能力** | 管道架构可迁移到其他系统 | → 数据库Agent | ✅ L2 |

### 1.2 专家视角

#### 专家共识

- **管道原则**：任何工具调用都不直接执行，必须先过权限管道
- **顺序原理**：Deny → Mode → Allow → Ask，每步有深刻理由
- **Bash特殊**：bash不是普通文本，是可执行动作描述
- **模式分离**：Plan=只读约束，Auto=安全自动过，Default=问用户
- **权限保护**：不仅保护恶意操作，还保护模型误判

#### 专家分歧

| 分歧 | 观点A | 观点B | 学习者立场 |
|------|-------|-------|-----------|
| **规则位置** | 配置文件 | 硬编码 | 高敏感硬编码 + 普通规则配置文件 |
| **检查层级** | Handler层 | Harness层 | Harness层统一管理 |
| **拒绝提示** | 只返回denied | 返回原因+建议 | 拒绝计数器 + 3次后建议切换模式 |
