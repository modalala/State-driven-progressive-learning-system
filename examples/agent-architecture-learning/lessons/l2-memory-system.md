## TODO 清单

### TODO-1: 概念辨析 - 记忆类型（🔴）

**目标**: 理解不同类型的记忆及其作用

**内容**:
1. 学习三类记忆的定义：
   - **工作记忆（Working Memory）**: 当前对话上下文，LLM 的 context window
   - **短期记忆（Short-term Memory）**: 当前会话内的历史记录，结构化存储
   - **长期记忆（Long-term Memory）**: 跨会话的知识，通常用向量数据库存储

2. 对比表格：
   | 特性 | 工作记忆 | 短期记忆 | 长期记忆 |
   |-----|---------|---------|---------|
   | 存储位置 | LLM Context | 内存/变量 | 向量数据库 |
   | 容量 | 有限（4K-128K tokens） | 中等 | 大 |
   | 持久性 | 当前请求 | 当前会话 | 永久 |
   | 检索方式 | 直接访问 | 索引访问 | 相似度搜索 |
   | 典型用途 | 当前推理 | 多轮对话 | 知识积累 |

3. 思考：为什么需要三层？不能只用一层吗？

**产出**:
- 对比表格
- 三层记忆必要性的分析

**完成检查**:
- [ ] 能准确区分三种记忆
- [ ] 能举例说明每种记忆的典型使用场景
- [ ] 理解为什么需要分层

### TODO-4: 记忆策略设计（🔴）

**目标**: 设计何时存、如何检索、何时清理的策略

**内容**:
1. **存储策略**（何时存入长期记忆）：
   - 所有 Observation？
   - 重要的 Thought？
   - 用户明确标记的内容？
   - 定期总结会话内容？

2. **检索策略**（如何决定检索什么）：
   - 基于当前 Query 的相似度搜索
   - 基于时间衰减（越新的越重要）
   - 基于重要性评分（用户标记或自动计算）
   - 混合策略：相似度 + 时间 + 重要性

3. **清理策略**（何时删除旧记忆）：
   - 固定容量（LRU）
   - 重要性低于阈值
   - 时间过期（如 30 天前的会话）
   - 自动总结压缩（保留摘要，删除详情）

4. 设计你的策略：

### TODO-5: 专家共识与分歧（🔴）

**目标**: 理解记忆系统设计的共识和争议

**内容**:
1. **共识提取**（5 个）：
   - 纯 LLM Context 不足以支撑复杂任务
   - 向量存储是当前语义记忆的主流方案
   - 记忆检索的质量直接影响 Agent 效果
   - 记忆管理需要权衡存储成本和检索效果
   - 不是所有信息都值得长期存储

2. **分歧探讨**（3 个）：
   - 分歧 A: 记忆应该结构化还是非结构化？
     - 结构化派：用知识图谱，关系明确
     - 非结构化派：纯文本向量，灵活简单
   
   - 分歧 B: 记忆检索应该主动还是被动？
     - 主动派：Agent 主动决定何时检索
     - 被动派：每次推理前自动检索相关记忆
   
   - 分歧 C: 跨 Agent 记忆共享是否可行？
     - 可行派：共享知识库加速学习
     - 隐私派：记忆是个性化的，不应共享

3. 选择立场

**产出**:
- 共识理解
- 分歧立场选择
