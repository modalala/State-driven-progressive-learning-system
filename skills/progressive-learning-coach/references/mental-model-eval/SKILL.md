# Mental Model Evaluation

评估心智模型理解层级，提供差距分析和完整答案。

## 理解层级框架

| 层级 | 名称 | 特征 | 评估标准 |
|------|------|------|----------|
| **L0** | 表面 | 记住事实，但不理解关联 | 能复述，无法解释"为什么" |
| **L1** | 关联 | 理解概念间因果关系 | 能解释"为什么"，能举例子 |
| **L2** | 深层 | 理解设计原则和抽象模式 | 能迁移到新情境，发现边界 |

---

## 评估流程

当用户回答深度测试问题后，按以下流程评估：

### 1. 判定层级

分析用户回答的特征：
- 是否只描述表面事实 → L0
- 是否解释因果关系 → L1
- 是否提炼设计原则 → L2

### 2. 差距分析

对比预期层级，指出缺失点：
- 用户回答中缺少的关键概念
- 用户回答中错误的理解
- 用户回答中未触及的深层原则

### 3. 完整答案

展示专家视角的 L2 理解：
- 专家共识
- 设计原则
- 可迁移模式
- 边界和例外

### 4. 改进建议

指导如何提升到下一层级：
- L0 → L1：问"为什么 X 能做到 Y"
- L1 → L2：提炼可迁移的设计原则

## 评估示例

### 问题：为什么加工具不需要改循环？

**用户回答**：「dispatch map 用字典分发，不用 if/elif」

**评估输出**：

---

**你的理解**: ⚠️ L0 层级（有 L1 趋势）

| 评估点 | 你的回答 | 达到层级 |
|--------|----------|----------|
| 表面事实 | "字典分发，不用 if/elif" | L0 ✅ |
| 因果关系 | 未解释"为什么能做到" | L0 ❌ |
| 设计原则 | 未提及开闭原则 | L0 ❌ |

---

**你理解的差距**：

❌ **缺失点 1**：只描述了代码结构，没解释因果
   - 你的回答："字典分发"
   - 应该说："循环通过 block.name 动态查表，不关心工具具体做什么"

❌ **缺失点 2**：没提炼出设计模式
   - 你的回答：未提及设计原则
   - 应该说："开闭原则——对扩展开放，对修改封闭"

---

**L2 层级完整理解**：

**核心原则**：**开闭原则** —— 对扩展开放（加工具），对修改封闭（循环不变）

**为什么能做到**：
- 循环只做一件事：检测 stop_reason，执行工具
- 工具名和参数都在 `tool_use` block 里，循环通过 `block.name` 动态查表
- `dispatch map` 把"是什么工具"和"怎么执行"解耦
- 加工具 = 加 entry + 加 schema，循环代码零改动

**可迁移模式**：
- 任何"注册 + 查表"机制都符合开闭原则
- 插件系统、策略模式、状态机同理
- 中间件链、过滤器链也是类似设计

**边界和例外**：
- 如果工具需要特殊的执行逻辑（如依赖其他工具结果），可能需要扩展循环
- 但这种情况应该通过工具组合解决，而不是改循环

---

**如何提升到 L2**：

1. 回答时先问自己："为什么 X 能做到 Y？"
2. 尝试提炼出可迁移的设计原则（不只是描述代码）
3. 思考：这个模式还能应用在哪里？

**练习问题**：
> 如果要加一个工具需要访问上一个工具的结果，你会怎么设计？循环需要改动吗？

## 关键原则

### 评估时必须做

1. **明确判定层级**：不只是说"L1"，要解释为什么判定为这个层级
2. **指出具体差距**：不只是说"不够深入"，要指出缺失的具体概念
3. **展示完整答案**：必须包含专家视角的 L2 理解
4. **给出改进建议**：告诉用户下一步怎么做

### 评估时不能做

1. **不能只说层级不给理由**
2. **不能只说差距不给完整答案**
3. **不能只给答案不给改进建议**
4. **不能模糊评估（如"差不多"）**
