# Method 4: Fast Learn（20分钟达标版）

## 核心目标

在20分钟内完成学习，达到：
- ✅ **L2理解层级**：理解设计原则和抽象模式
- ✅ **用自己的话定义**：不依赖原文词汇
- ✅ **暴露理解盲区**：通过对抗测试发现知识鲁棒性问题

---

## 设计原则

| 原阶段 | 优先级 | 理由 | 精简策略 |
|--------|--------|------|----------|
| Phase 1 (心智模型) | ⭐⭐⭐ 核心 | L2前提是概念网络建立 | **保留全部**：核心概念表 + 深度问题 |
| Phase 3 (对抗测试) | ⭐⭐⭐ 核心 | 只有对抗测试能暴露盲区 | **保留全部**：反事实情境 |
| Phase 2 (SQ3R) | ⭐ 一般 | 过程方法论可精简 | **只保留Read**：5分钟快速读关键段落 |

**核心洞察**：Phase 1+3 融合，Phase 2 只做 Read。

---

## 执行流程（20分钟）

---

### Phase 0: 核心概念提取与选择（启动时必须执行）

**触发时机**：用户请求Fast Learn时，必须先执行此步骤。

---

#### Step 1: 提取核心概念

**教练行为**：
1. 读取学习材料全文
2. 按文章结构提取所有核心概念
3. 记录每个概念的来源位置

**输出格式**：

```markdown
## 核心概念提取结果

| 来源 | 概念 | 初步判断 |
|------|------|----------|
| [章节/位置] | [概念1] | [简要说明] |
| [章节/位置] | [概念2] | [简要说明] |
| ... | ... | ... |
```

---

#### Step 2: 重要性排序

**判断标准**：

| 标准 | 权重 | 说明 |
|------|------|------|
| **理解前提性** | 高 | 是否理解其他概念的前提 |
| **角色影响性** | 高 | 是否直接影响工作定位 |
| **实用性** | 中 | 明天能否直接应用 |
| **抽象程度** | 低 | 抽象概念排序靠后 |
| **专项领域** | 低 | 安全/经济等专项后置 |

**输出格式**：

```markdown
## 重要性排序

| 排序 | 概念 | 重要性理由 | 影响 |
|------|------|------------|------|
| **#1** | [概念] | [理由] | [影响范围] |
| **#2** | [概念] | [理由] | [影响范围] |
| ... | ... | ... | ... |
```

---

#### Step 3: 数量判断与方案选择

**判断规则**：

| 概念数量 | 动作 |
|----------|------|
| **≤5个** | 直接进入Section A，使用全部概念 |
| **>5个** | 列出排序，让用户选择方案 |

**超过5个时的输出格式**：

```markdown
## 判断结果

**核心概念 > 5个，需要用户选择方案**

---

## 请选择学习方案

| 方案 | 包含概念 | 预估时间 |
|------|----------|----------|
| **方案A：AI精选5个（推荐）** | [列出前5个概念名] | 20分钟 |
| **方案B：完整学习** | [概念总数]个概念 | [N*4分钟]分钟 |
| **方案C：用户自选** | 你选择想深入的概念（从[N]个中选） | 根据选择 |
| **方案D：AI精选+用户追加** | 先学5个核心，再追加用户关心的概念 | 20分钟+追加时间 |

---

请选择方案，或告诉我你想增加哪些概念。
```

---

#### 方案执行规则

| 方案 | 执行方式 |
|------|----------|
| **方案A** | 使用排序前5的概念，标准20分钟流程 |
| **方案B** | 使用全部概念，每个概念4分钟，总时间按概念数计算 |
| **方案C** | 用户指定概念列表，按用户选择执行 |
| **方案D** | 先执行前5概念（20分钟），完成后询问是否追加 |

---

#### 必须做

| 序号 | 规则 |
|------|------|
| 1 | **必须先提取概念**，不能直接进入Section A |
| 2 | **必须排序**，按重要性排序而非随机 |
| 3 | **超过5个必须询问**，不能自行删减 |
| 4 | **排序理由要说明**，让用户理解为什么这样排序 |

---

#### 必须不做

| 序号 | 禁止行为 |
|------|----------|
| 1 | ❌ 不经提取直接列5个概念 |
| 2 | ❌ 不排序直接让用户选 |
| 3 | ❌ 超过5个不询问，自行删减 |
| 4 | ❌ 不说明排序理由 |

---

### Section A: 核心概念用自己的话定义（每个概念约4分钟）

**执行方式**：不展示原文定义，让学生用自己的话填写。

**输出模板**（根据Phase 0选择的概念动态生成）：

```markdown
| 概念 | 你的定义（不用原文词汇） | 自评（L0/L1/L2） |
|------|--------------------------|------------------|
| [概念1] | [学生填写] | |
| [概念2] | [学生填写] | |
| [概念3] | [学生填写] | |
| [概念4] | [学生填写] | |
| [概念5] | [学生填写] | |
| ...（根据选择数量） | | |
```

**L2达标标准**：
- 定义准确（意思对）
- 用自己的词（不是抄原文）
- 提到"为什么"（深层原因）

**教练行为**：
1. 展示概念列表，不展示原文定义
2. 等待学生填写
3. 填写完成后，逐一评估：
   - ✅ 准确 + 自己的词 + 有原因 → L2
   - ⚠️ 准确但无原因 → L1
   - ❌ 不准确或有原文词汇 → L0
4. 对L0/L1的回答给出差距分析和改进方向

---

### Section B: L2理解关键点（5分钟）

**执行方式**：展示"表面→深层"理解链，让学生识别自己的理解层级。

**输出模板**：

```markdown
## 三组"表面→深层"理解链

| 问题 | L0表面回答 | L2深层回答（你需要达到的） |
|------|-----------|---------------------------|
| [关键问题1] | [表面描述] | + [深层原因1] + [深层原因2] → [抽象模式] |
| [关键问题2] | [表面描述] | + [深层原因] → [设计原则/边界] |
| [关键问题3] | [表面描述] | + [深层原因] → [可迁移模式] |
```

**教练行为**：
1. 展示三组理解链
2. 让学生对比自己的理解，识别差距
3. 对差距大的项，补充专家视角解释

---

### Section C: 对抗测试暴露盲区（10分钟）

**执行方式**：三个反事实情境，暴露理解边界。

**输出模板**：

```markdown
## 反事实情境测试

> **Q1**: [情境1 - 测试核心概念边界]
>
> **你的回答**：

> **Q2**: [情境2 - 测试触发/失效条件]
>
> **你的回答**：

> **Q3**: [情境3 - 测试极端设计决策]
>
> **你的回答**：
```

**教练行为**：
1. 逐一提出情境问题
2. 学生回答后，追问"为什么这样思考"
3. 发现漏洞时：
   - 不直接否定
   - 通过追问引导发现
   - 展示专家视角的完整理解
4. 记录盲区：

```markdown
## 盲区记录

| 盲区 | 来源问题 | 学生错误理解 | 正确理解 |
|------|----------|--------------|----------|
| [盲区1] | Q1 | | |
| [盲区2] | Q2 | | |
```

---

## 评估输出（结束时）

```markdown
## Fast Learn 完成

### 达标情况
- 概念定义：[X/N达到L2]（N为选择的概念数量）
- 理解关键点：[差距项]
- 对抗测试：[通过X/3题]

### 已建立的核心心智模型
- [学生正确理解的核心点1]
- [学生正确理解的核心点2]

### 需要关注的盲区
- [盲区1]：[补救方向]
- [盲区2]：[补救方向]

### 学习状态
- 如果盲区≤1：✅ 完成，进入复习计划
- 如果盲区≥2：⚠️ 建议补充学习，再重新测试
```

---

## 反事实情境设计原则

| 测试目标 | 情境设计要点 |
|----------|--------------|
| 核心概念边界 | 去掉一个必要元素，问"还算不算" |
| 触发/失效条件 | 描述一个看似匹配但实际失效的场景 |
| 极端设计决策 | 描述一个违反原则但看似合理的做法 |

**示例（Claude Skills）**：

```
Q1: Skill没有验收标准，流程详细，算不算"好的Skill"？
    → 测试"验收必须可执行"原则的边界

Q2: 用户说"代码审查"，Skill未触发，最可能原因？
    → 测试description作为路由规则的失效条件

Q3: 10个提示词放进一个SKILL.md，有什么问题？
    → 测试入口文件长度与渐进式披露原则
```

## 与其他方法的关系

| 方法 | 时间 | 适用场景 |
|------|------|----------|
| Method 1-3 完整版 | 60-90分钟 | 系统学习，建立完整心智模型 |
| **Method 4 Fast Learn** | 20分钟 | 快速学习，暴露盲区后决定是否深入 |

**决策树**：
```
用户请求Fast Learn
    │
    ▼
Phase 0: 提取概念 → 排序
    │
    ├─→ ≤5个 → 直接进入Section A
    │
    └─→ >5个 → 用户选择方案
              │
              ├─→ 方案A（AI精选5个） → 20分钟流程
              ├─→ 方案B（完整学习） → N*4分钟
              ├─→ 方案C（用户自选） → 按选择执行
              └─→ 方案D（AI+追加） → 20分钟+追加
              │
              ▼
          Section A → Section B → Section C
              │
              ▼
          盲区≤1 → 完成，复习计划
          盲区≥2 → 建议 Method 1-3 完整版
```

---

## 关键约束

| 约束 | 原因 |
|------|------|
| **Phase 0 必须先执行** | 提取概念是后续一切的基础 |
| **超过5个概念必须询问用户** | 尊重用户选择，不自行删减 |
| **概念排序必须说明理由** | 让用户理解排序逻辑 |
| Section A 不展示原文定义 | 强迫用自己的话，检验真实理解 |
| Section C 必须追问推理过程 | 检验是背诵还是理解 |
| 不跳过对抗测试 | 盲区暴露是本方法的核心价值 |
| 时间动态调整 | 根据概念数量调整，每个概念约4分钟 |

---

## 教练行为准则

### 必须做

1. **先问后教**：Section A 让学生先填写
2. **追问推理**：Section C 问"为什么这样思考"
3. **记录盲区**：每个漏洞都要记录
4. **给出补救方向**：不只指出问题，还给出方向

### 不能做

1. 不直接给出答案，让学生先表达
2. 不评判立场选择，引导深入思考
3. 不跳过追问环节，即使回答看起来正确
4. 不模糊评估（如"差不多"), 必须明确层级
